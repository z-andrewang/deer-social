## 4. 认证与状态管理

### 4.1 多服务 API 配置

前端需要对接两个独立后端服务，配置两个 axios 实例：

```typescript
// lib/api.ts
export const identityApi = axios.create({
  baseURL: process.env.NEXT_PUBLIC_IDENTITY_API_URL,  // e.g. https://identity.deer-social.com
  withCredentials: true,  // 自动携带 HttpOnly Cookie
});

export const businessApi = axios.create({
  baseURL: process.env.NEXT_PUBLIC_BUSINESS_API_URL,  // e.g. https://api.deer-social.com
  withCredentials: true,
});

// 统一 401 拦截：Token 过期时先请求 Identity 刷新，再重试原请求
businessApi.interceptors.response.use(
  (res) => res,
  async (err) => {
    if (err.response?.status === 401) {
      await identityApi.post('/auth/refresh');
      return businessApi.request(err.config);
    }
    return Promise.reject(err);
  }
);
```

### 4.2 Cookie 策略

两个服务共享同一域名（或子域名）下的 Cookie，前端**不直接操作 Token**：

- 登录/注册请求发往 **Identity**，成功后浏览器自动携带 Cookie
- 业务请求发往 **Business**，Cookie 由浏览器自动附加（需两个服务配置相同的 Cookie Domain）
- Token 续期由 **Identity** 在响应中通过 `Set-Cookie` 刷新

### 4.3 全局状态（Zustand）

```typescript
interface AppState {
  account: Account | null;     // Identity 返回的账户信息（id, email, isVerified, role）
  profile: Profile | null;     // Business 返回的业务 Profile（nickname, avatar, mbti, tags, bio）
  isAuthLoading: boolean;      // 认证状态加载中
  unreadCount: number;         // 未读消息总数
  dailyLimit: {                // 当日匹配操作剩余次数
    remaining: number;
    total: number;
  };
}
```

### 4.4 数据获取（SWR）

- **账户信息**：`useSWR('/identity/me', () => identityApi.get('/identity/me'))` — 仅返回 `id, email, isVerified, role`
- **业务 Profile**：`useSWR('/profile/me', () => businessApi.get('/profile/me'))` — 返回 `nickname, avatar, mbti, tags, bio`
- **推荐流**：`useSWR('/posts/feed', () => businessApi.get('/posts?type=feed'))`
- **关注流**：`useSWR('/posts/following', () => businessApi.get('/posts?type=following'))`
- **消息列表**：`useSWR('/conversations', () => businessApi.get('/chat/conversations'))`
- **匹配推荐**：`useSWR('/match/recommend', () => businessApi.get('/match/recommend'))`

## 5. 各模块前端实现

### 5.1 账户中心 (Identity)

- **注册/登录表单**：React Hook Form + Zod 校验，邮箱格式校验、密码强度提示
- **登录后流程**：
  1. 调用 `identityApi.post('/auth/login')` 获取账户凭证（Cookie 自动设置）
  2. 调用 `businessApi.get('/profile/me')` 获取业务 Profile（不存在时自动创建默认记录）
- **邮箱验证**：点击验证邮件后跳转回应用，前端轮询验证状态或等待用户手动刷新

### 5.2 Profile 管理（Business）

- **Profile 编辑**：支持昵称、头像（Supabase Storage 上传）、MBTI 自填选择器、兴趣标签输入（逗号分隔或 Tag 组件）、个人简介
- **API 归属**：所有 Profile 操作通过 `businessApi` 调用 `/profile/me` 或 `/profile/:userId`

### 5.3 社区讨论 (PDB)

- **帖子卡片**：展示作者头像/昵称、内容、多图 Grid、点赞/评论计数、发布时间
- **发帖器**：富文本简化版（纯文本 + 多图上传），提交后进入"待审核"状态提示
- **信息流**：无限滚动（Intersection Observer + 分页加载），支持下拉刷新
- **热度排序**：前端不计算，纯展示后端返回的排序结果
- **二级评论**：树形嵌套展示，一级评论下展开回复列表

### 5.4 匹配社交 (Match)

- **用户卡片**：
  - 顶部：头像、昵称、MBTI 类型（含认证标识，如 `INTJ · 中级认证`）
  - 中部：兴趣标签、个人简介
  - 底部：三按钮操作栏
- **三按钮交互**：
  - **喜欢**：调用 `POST /match/like`，成功后方卡片滑出，下一张卡片进入
  - **不喜欢**：调用 `POST /match/pass`，同上
  - **主动发一句话**：弹出输入框，输入招呼语后调用 `POST /match/greet`，发送成功后自动进入私聊页面或提示已发送
- **每日次数限制**：操作前本地校验剩余次数，耗尽后按钮置灰并提示"今日次数已用完"
- **Match 成功弹窗**：双向喜欢时弹出全屏动画 + "你们已互相喜欢，开始聊天吧"引导

### 5.5 即时通讯 (IM)

- **会话列表**：展示最近联系人、最后一条消息预览、未读数角标
- **聊天窗口**：
  - 消息气泡（左/右对齐）
  - 输入框 + 发送按钮
  - 敏感词命中时输入框下方实时提示"消息包含敏感内容，无法发送"
  - 首次对话顶部显示防诈骗提示横幅
  - 消息长按/右键菜单：撤回（2分钟内）
- **实时推送**：
  - 进入聊天室后，通过 `supabase.realtime` 订阅 `messages:conversation_id=eq.xxx`
  - 收到新消息时，SWR 缓存增量更新，未读数 +1
  - 离线时消息由服务端保留，重新上线后通过 Realtime 补发

### 5.6 MBTI 测试与认证

- **答题界面**：
  - 单题全屏展示，选项为 A/B 二选一（或情景多选）
  - 顶部进度条显示当前题号/总题数
  - 支持"上一题"回退修改
  - 中途退出时提示"退出后本次测试不保存"
- **结果页**：
  - 展示匹配出的 MBTI 类型
  - 展示认证等级标识（初级/中级/高级）
  - 各维度雷达图或条形图（E-I、S-N、T-F、J-P 得分分布）
  - 提供"保存到 Profile"和"重新测试"按钮
- **Profile 页展示**：MBTI 类型右侧展示认证徽章或"自填"标签

### 5.7 管理后台 (Admin)

- **技术栈**：React + Refine（Ant Design 风格），独立部署于 `apps/admin`
- **核心页面**：
  - 帖子审核列表（待审/已通过/已拒绝筛选）
  - 用户管理列表（封禁/解封操作）
  - 审计日志（只读表格，支持时间范围筛选）
  - MBTI 题库管理（题目增删改查，二期可扩展）

## 6. 性能优化

| 策略 | 实现方式 |
|------|---------|
| 图片优化 | Next.js `<Image>` 组件，自动 WebP 转换、懒加载、响应式尺寸 |
| 代码分割 | Next.js 自动按路由分割，`dynamic import` 处理重型组件（图表库） |
| 数据缓存 | SWR `stale-while-revalidate`，减少重复请求 |
| 实时连接管理 | 页面离开聊天室时取消 Supabase Realtime 订阅，避免内存泄漏 |
| 静态资源缓存 | Cloudflare Page Rules 强缓存 JS/CSS/图片，CDN 边缘节点分发 |

## 7. 大陆访问优化

- **Cloudflare 代理**：Vercel 域名开启 CF 代理，利用优选 IP 改善中国大陆访问质量
- **Supabase 静态资源**：头像/帖子图片通过 Cloudflare 缓存，减少直接回源 Supabase Storage
- **Realtime 降级预留**：MVP 阶段纯依赖 WebSocket，若大陆环境下稳定性不足，二期评估引入短轮询兜底（技术债务 R-01）
