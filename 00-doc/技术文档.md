# 技术文档

## 1. 总体架构图

系统采用 **Monorepo** 组织代码，前后端分离，利用 **Supabase** 处理数据层，**Vercel** 进行边缘分发。

## 2. 目录结构 (Turborepo)

```
/deer-social
├── apps
│   ├── web              # 响应式前端 (Next.js 14) - 包含社区、社交、游戏
│   ├── admin            # 后台管理系统 (React + Refine)
│   └── server           # 核心后端 (NestJS)
├── packages
│   ├── database         # Prisma Schema & Client (Neon/PostgreSQL)
│   ├── ui               # 共享 UI 组件库 (Tailwind + Shadcn)
│   ├── types            # 共享 TS 定义 (API 接口定义)
│   └── config           # 共享 Eslint/Tailwind 配置
└── turbo.json
```

## 4. 关键技术方案实现

### 4.1 账户中心与 SSO 模拟

- **实现方式**：NestJS 使用 Passport-JWT。
- **流程**：
  1. 用户在 web 应用登录；
  2. 后端返回 `access_token`；
  3. 存储在 HttpOnly Cookie 或域名共享的 LocalStorage 中。
- **多应用识别**：在 JWT Payload 中加入应用标识符，后端根据标识符返回不同的应用权限。

### 4.2 响应式 UI 适配

- **策略**：使用 Tailwind CSS 的断点系统。
- **布局**：
  - **Mobile**：全屏信息流，底部 Tab Bar 导航。
  - **PC/Tablet**：左右三栏布局（左侧导航、中间信息流、右侧排行榜/匹配卡片），内容区域最大宽度限制在 **1200px**。

### 4.3 实时私聊 (IM)

- **技术**：Supabase Realtime。
- **逻辑**：
  1. 用户进入聊天室，订阅 `messages:chat_id=eq.xxx` 频道。
  2. 发送消息通过 NestJS API 写入 PostgreSQL。
  3. Supabase 捕获数据库变更，秒级推送到接收方前端。

### 4.4 大陆访问优化

- **CDN**：Cloudflare 设置 Page Rules，对静态资源和 Supabase 存储地址进行强缓存。
- **API 代理**：Vercel 部署地址绑定你的域名，开启 CF 代理，通过优选 IP 绕过默认节点的拥堵。

## 5. 部署流水线 (CI/CD)

| 环节 | 说明 |
|------|------|
| 代码提交 | GitHub Push 到 `main` 分支 |
| 前端部署 | Vercel 自动识别 `apps/web` 进行构建 |
| 后端部署 | Railway 自动识别 `apps/server` 进行容器化部署 |
| 数据库 | Neon 自动处理 Migrations |
