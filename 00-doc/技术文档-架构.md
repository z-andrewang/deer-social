## 1. 总体架构图

系统采用 **Monorepo** 组织代码，前后端分离：

- **数据层**：Supabase（PostgreSQL + Realtime + Storage）
- **边缘分发**：Vercel（前端托管 + Edge Functions）
- **后端服务**：Railway（Identity + Business 双服务容器化部署）
- **CDN/代理**：Cloudflare（静态资源缓存 + 优选 IP）

## 2. 目录结构 (Turborepo)

```
/deer-social
├── apps
│   ├── web              # 响应式前端 (Next.js 14 App Router)
│   ├── admin            # 后台管理系统 (React + Refine)
│   ├── identity         # 用户中心服务 (NestJS) — 注册/登录/认证/Profile 基础管理
│   └── business         # 业务服务 (NestJS) — 社区/匹配/聊天/MBTI/管理后台
├── packages
│   ├── database         # Prisma Schema & Client (Neon/PostgreSQL)
│   ├── ui               # 共享 UI 组件库 (Tailwind + Shadcn)
│   ├── types            # 共享 TS 定义 (API 接口类型)
│   └── config           # 共享 ESLint / Tailwind / TS 配置
└── turbo.json
```

## 3. 数据库设计 (ER Diagram)

### 3.1 核心表结构

| 表名 | 字段 | 说明 |
|------|------|------|
| `users` | `id`, `email`, `password_hash`, `is_verified`, `role`, `created_at` | 账户核心表，由 Identity Service **独占写入**，Business 只读 |
| `profiles` | `user_id`, `nickname`, `avatar_url`, `mbti_type`, `mbti_cert_level` (`NULL`/`basic`/`intermediate`/`advanced`), `bio`, `tags[]`, `updated_at` | 业务身份卡片，由 Business Service 管理（不同业务可自定义字段） |
| `posts` | `id`, `author_id`, `content`, `images[]`, `status` (`PENDING`/`APPROVED`/`REJECTED`), `reject_reason`, `like_count`, `comment_count`, `created_at` | 社区帖子，需审核后公开 |
| `post_likes` | `post_id`, `user_id`, `created_at` | 帖子点赞关联表（复合主键防重复） |
| `comments` | `id`, `post_id`, `author_id`, `parent_id` (`NULL` 为一级评论), `content`, `created_at` | 二级评论系统 |
| `follows` | `follower_id`, `following_id`, `created_at` | 关注关系（复合主键） |
| `matches` | `id`, `user_one_id`, `user_two_id`, `status` (`PENDING`/`MATCHED`), `created_at` | 双向喜欢后建立的社交关系 |
| `conversations` | `id`, `user_one_id`, `user_two_id`, `created_at` | 私聊会话（Match 或主动发一句话后建立） |
| `messages` | `id`, `conversation_id`, `sender_id`, `content`, `is_deleted`, `original_content`, `created_at` | 消息表，支持撤回（软删除） |
| `mbti_questions` | `id`, `level` (`basic`/`intermediate`/`advanced`), `dimension`, `content`, `options[]`, `weights[]`, `order` | MBTI 题库 |
| `mbti_test_results` | `id`, `user_id`, `level`, `mbti_type`, `dimension_scores` (JSON), `duration_seconds`, `created_at` | 用户测试记录 |
| `audit_logs` | `id`, `actor_id`, `action`, `target_type`, `target_id`, `details` (JSON), `created_at` | 管理员操作审计日志，不可修改删除 |
| `game_scores` | `id`, `user_id`, `game_id`, `score`, `updated_at` | 游戏分数（二期） |

### 3.2 关键索引建议

| 表 | 索引字段 | 用途 |
|----|---------|------|
| `posts` | `(status, created_at DESC)` | 推荐流/关注流筛选 |
| `post_likes` | `(post_id, user_id)` | 点赞去重与计数 |
| `comments` | `(post_id, parent_id, created_at)` | 帖子评论列表 |
| `follows` | `(follower_id)`, `(following_id)` | 关注/粉丝列表查询 |
| `matches` | `(user_one_id, user_two_id)` | 双向查询 Match 关系 |
| `messages` | `(conversation_id, created_at DESC)` | 消息列表分页 |
| `mbti_test_results` | `(user_id, level, created_at DESC)` | 查询用户最高等级认证 |

## 4. 服务边界与通信

| 通信方向 | 协议/方式 | 说明 |
|----------|----------|------|
| Web ↔ Identity | HTTPS / REST JSON | 前端调用用户中心接口（注册/登录/Token 刷新/账户信息查询） |
| Web ↔ Business | HTTPS / REST JSON | 前端调用业务接口（Profile/社区/匹配/聊天/MBTI） |
| Web ↔ Supabase | HTTPS / WebSocket | 实时消息通过 Supabase Realtime 订阅 |
| Business ↔ Identity | HTTPS / REST JSON 或 共享数据库 | Business 验证 Token 时可调用 Identity 验证接口，或共享 JWT Secret 本地校验 |
| Identity / Business ↔ Neon | TCP / PostgreSQL | Prisma Client 连接数据库（Identity 独占 `users` 表写入） |
| Identity / Business ↔ Redis | TCP | 热帖缓存、会话黑名单、每日操作次数计数 |
| Admin ↔ Business | HTTPS / REST JSON | 后台管理系统调用业务服务 Admin 模块 |

## 5. 部署架构

| 组件 | 平台 | 说明 |
|------|------|------|
| 前端 (web) | Vercel | 自动识别 `apps/web`，支持 SSR/Edge |
| 管理后台 (admin) | Vercel | 独立部署，独立域名或子路径 |
| 用户中心 (identity) | Railway | 自动识别 `apps/identity`，独立容器化部署，独立域名 |
| 业务服务 (business) | Railway | 自动识别 `apps/business`，独立容器化部署，独立域名 |
| 数据库 | Neon | Serverless PostgreSQL，两个服务共享同一实例，Schema 级别隔离 |
| 实时通信 | Supabase | Realtime 服务，与 Neon 数据层联动 |
| 对象存储 | Supabase Storage | 头像、帖子图片存储，通过 CDN 加速 |
| 缓存 | Upstash Redis / Vercel KV | 热帖缓存、计数器、Token 黑名单（两服务共享） |

## 6. CI/CD 流水线

| 环节 | 触发条件 | 平台 | 行为 |
|------|---------|------|------|
| 代码提交 | GitHub Push 到 `main` 分支 | GitHub Actions | 执行 Lint / Type Check / Test |
| 前端构建 | `apps/web` 代码变更 | Vercel | 自动构建并部署，支持预览环境 |
| 后台构建 | `apps/admin` 代码变更 | Vercel | 自动构建并部署 |
| 用户中心构建 | `apps/identity` 代码变更 | Railway | 自动构建 Docker 镜像并滚动部署 |
| 业务服务构建 | `apps/business` 代码变更 | Railway | 自动构建 Docker 镜像并滚动部署 |
| 数据库迁移 | `packages/database` Prisma Schema 变更 | GitHub Actions | 执行 `prisma migrate deploy` 到 Neon（Identity 与 Business 共用） |

## 7. 安全架构概览

- **传输层**：全站 HTTPS，Cookie 配置 `Secure`、`HttpOnly`、`SameSite=Lax/Strict`
- **认证层**：Passport-JWT，Access Token 48h Idle Timeout + Refresh Token 2h
- **密码存储**：bcrypt / argon2 哈希，禁止明文
- **上传限制**：JPG/PNG/GIF 格式，单文件 ≤ 5MB
- **审计**：管理员操作全量记录审计日志，不可篡改
